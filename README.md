# PFC NOTE

食事の写真を撮るだけで、AIがタンパク質・脂質・炭水化物（PFC）を自動計算してくれる、個人開発の食事記録Webアプリです。

サーバーを一切持たない「単一HTMLファイル構成」で、自分のブラウザとAI APIキーだけで動作します。

---

## 🎯 このアプリで解決したいこと

市販の食事記録アプリの多くは、広告・アカウント登録・サブスクリプションが必要だったり、写真データがどこかのサーバーに保存され続けることがあります。
そのため、PFC NOTEという食事記録Webアプリを作成しました。

以下を重要視して作成

- **写真は解析のためだけに使い、保存しない**（プライバシー配慮とアプリの軽量化）
- **自分のAPIキーで動かす（BYOK）ので、サーバー運用コストがゼロ**
- **無料枠だけで実用十分**（Google Gemini APIを利用）
- **他人のアカウントに依存しない**、自分だけのデータ

## ✨ 主な機能

| 機能 | 説明 |
|---|---|
| 📷 写真からPFC自動解析 | 食事を撮影するとAIが栄養素を推定 |
| 🧩 品目の内訳合算 | 品目ごとに写真を分けて撮影しても、1食分として自動で合算 |
| ✍️ 手動入力 | 写真なしでも数値を直接入力可能 |
| 🤖 AI再解析 | 食材の補足情報を追記して精度アップ |
| 💬 AIチャット相談 | 栄養に関する質問をAIが回答 |
| 📅 カレンダー表示 | 月単位・年単位で摂取履歴を閲覧可 |
| 🔥 ヒートマップ | 年間ビューでカロリー傾向を可視化 |
| 🔌 マルチAIプロバイダー対応 | Google Gemini（無料）/ Anthropic Claude（有料）を切り替え可能 |

## 🛠 技術スタック

- **HTML / CSS / JavaScript（Vanilla、フレームワーク不使用）**
- **Google Fonts**（Zen Kaku Gothic New / Noto Sans JP / JetBrains Mono）
- **Google Gemini API**（`generateContent` エンドポイントへブラウザから直接リクエスト）
- **Anthropic Claude API**（同上、選択式）
- **localStorage**（記録データの永続化、サーバー不使用）
- **GitHub Pages**（無料ホスティング）

外部ライブラリやビルドツールは一切使用していません。HTMLファイル1つをブラウザで開くだけで動作します。

## 🏗 設計のポイント

- **単一ファイル・ゼロ依存**：`index.html` 1枚で完結。npm installもビルドも不要で、誰でもすぐに動かせる
- **BYOK（Bring Your Own Key）方式**：APIキーをユーザー自身が発行・管理するため、開発者側の運用コストとセキュリティリスクがゼロ
- **プロバイダー抽象化**：AI呼び出し部分を関数単位で分離し、Gemini / Anthropic のどちらでも同じUIから使えるよう設計
- **プライバシーバイデザイン**：画像データはAI解析リクエストの送信にのみ使用し、一切ローカル・外部に保存しない

## 💾 容量・パフォーマンスについて

画像を保存しない設計のため、長期間使い続けてもデータ量・動作速度への影響はほぼありません。

- 1日分のデータ量：約500バイト（食事名とPFC数値のみ）
- 10年間毎日記録し続けても：**約1.8MB程度**（`localStorage`の上限5〜10MBに対して十分小さい）
- 年間ビュー（最大366件のデータ読み込み・集計）も、体感できないレベル（数ミリ秒）で完了
- 一度表示した月・年のデータはキャッシュされ、再表示時の再取得も発生しない

## 🚀 使い方

1. このリポジトリを Fork または `index.html` をダウンロード
2. GitHub Pages などの静的ホスティングで公開（あるいはローカルで開くだけでもOK）
3. [Google AI Studio](https://aistudio.google.com/apikey) で無料のAPIキーを発行
4. アプリ右上の⚙設定からAPIキーを登録
5. 食事を記録開始
